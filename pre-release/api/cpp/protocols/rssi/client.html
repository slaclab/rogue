Client

Client is the role-specific convenience wrapper that owns and wires the RSSI Transport, Application, and Controller objects for client operation. For conceptual guidance, see RSSI Protocol Client.

Threading and Lifecycle

  • Delegates protocol-state execution to Controller and does not create a dedicated Client worker thread.

  • Implements Managed Interface Lifecycle: Example: Runtime Work In _start() And _stop()

Python binding

This C++ class is also exported into Python as rogue.protocols.rssi.Client.

Python API page: - Client

objects in C++ are referenced by the following shared pointer typedef:

typedef std::shared_ptr<rogue::protocols::rssi::Client> rogue::protocols::rssi::ClientPtr
[header]

The class description is shown below:

class Client
[header]

RSSI client convenience wrapper.

Bundles transport, application, and controller components for client-side RSSI links. Protocol reference: https://confluence.slac.stanford.edu/x/1IyfD

Construction wires the internal components together: Transport <-> Controller <-> Application.
